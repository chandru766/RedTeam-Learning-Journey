## 🧠 What I Learned

### 🛡️ Cybersecurity

* The practice of protecting systems, networks, and data from cyber attacks.

#### ⚙️ How It Works:

* Uses technologies, processes, and policies to defend digital assets.

---

### ⚠️ Threat

* Anything that can cause harm to a system, network, or organization.

#### Examples:

* Hackers
* Malware
* Insider threats
* Natural disasters

---

### 🕳️ Vulnerability

* A weakness in a system that can be exploited by an attacker.

#### Examples:

* Weak passwords
* Unpatched software
* Misconfigured services

---

### 📉 Risk

* The likelihood that a threat will exploit a vulnerability and cause damage.

#### Formula:

```text id="rsk001"
Risk = Threat × Vulnerability × Impact
```

---

### 💥 Exploit

* A technique or code used to take advantage of a vulnerability.

#### Example:

* Exploiting an outdated application to gain unauthorized access.

---

### 🎯 Payload

* The action or code delivered after successful exploitation.

#### Examples:

* Reverse shell
* Malware
* Data theft tools

---

### 🌐 Attack Surface

* All possible entry points that an attacker can target.

#### Examples:

* Open ports
* Web applications
* User accounts
* Public-facing services

---

### 🔐 Authentication

* Verifying the identity of a user or system.

#### Examples:

* Username and password
* Multi-Factor Authentication (MFA)
* Biometrics

---

### 🔑 Authorization

* Determining what an authenticated user is allowed to access.

#### Example:

* Admin users have more permissions than standard users.

---

### 🏛️ CIA Triad

#### 🔒 Confidentiality

* Ensures data is accessible only to authorized users.

#### ✅ Integrity

* Ensures data remains accurate and unaltered.

#### ⚡ Availability

* Ensures systems and data are available when needed.

---

### 🦠 Malware

* Malicious software designed to damage or compromise systems.

#### Types:

* Virus
* Worm
* Trojan
* Ransomware
* Spyware

---

### 🎣 Phishing

* A social engineering attack used to steal sensitive information.

#### Example:

* Fake login pages
* Fraudulent emails

---

### 🚨 Zero-Day Vulnerability

* A security flaw unknown to the vendor or without an available patch.

#### Why It's Dangerous:

* Attackers can exploit it before a fix is released.

---

### 🚑 Incident Response

* The process of identifying, containing, and recovering from security incidents.

#### Phases:

1. Preparation
2. Detection
3. Containment
4. Eradication
5. Recovery
6. Lessons Learned

---

## 💡 Key Takeaways

* Cybersecurity is about protecting systems, networks, and data
* Vulnerabilities create opportunities for attackers
* Risks arise when threats exploit vulnerabilities
* Authentication verifies identity, while authorization controls access
* The CIA Triad is the foundation of information security
* Understanding cybersecurity terminology is essential for every security professional
