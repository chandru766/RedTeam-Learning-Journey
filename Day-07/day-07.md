## 🧠 What I Learned

### 🛠️ What is Penetration Testing

* Penetration Testing (Pentesting) is a **simulated cyber attack** on a system
* is to find vulnerabilities before attackers do

👉 Done with permission and in a controlled environment

---

### ⚖️ Software Testing vs Security Testing

* **Software Testing**
  → Checks if application works correctly (bugs, errors)

* **Penetration Testing**
  → Finds security vulnerabilities and weaknesses

👉 One focuses on functionality, the other on security

---

### ⚙️ How Penetration Testing Works

1. **Reconnaissance (Information Gathering)**
2. **Scanning**
3. **Gaining Access (Exploitation)**
4. **Maintaining Access**
5. **Reporting**

👉 Follows a structured approach

---

### 🎯 Types of Penetration Testing

* **Black Box Testing**
  → No prior knowledge

* **White Box Testing**
  → Full access and information

* **Gray Box Testing**
  → Partial knowledge

---

### 🧑‍💻 Ethical Hacker vs Penetration Tester

* **Ethical Hacker**
  → Broad role, focuses on security practices

* **Penetration Tester**
  → Specific role, performs authorized attacks

👉 Both aim to improve security

---

### 🔍 OSINT Framework

* OSINT = **Open Source Intelligence**
* Collecting information from publicly available sources

Examples:

* Social media
* Websites
* Public records

👉 Used in reconnaissance phase
