## 🧠 What I Learned

### 🔢 Logical Ports

* Logical ports are **virtual communication endpoints** in a system
* They allow multiple applications to communicate over a network
* Works at **Transport Layer (Layer 4)**

👉 Not physical — they exist in software

---

### 📡 Protocols

Protocols define rules for communication between devices

#### 🌐 Web & Transfer Protocols:

* HTTP / HTTPS → Web browsing
* FTP / SFTP → File transfer

#### 🔐 Remote Access:

* SSH → Secure remote login
* Telnet → Unsecure remote login

#### 📧 Email Protocols:

* SMTP → Sending emails
* POP3 → Receiving emails (download)
* IMAP → Receiving emails (sync)

#### 🌍 Network & Directory:

* DNS → Domain name resolution
* DHCP → Automatic IP assignment
* LDAP → Directory services

#### 📊 Monitoring & Management:

* SNMP → Network monitoring

#### 🗄️ Database Protocols:

* SQL Server → Database communication
* Oracle Database → Enterprise database systems

👉 Protocols act like **rules/languages** for communication

---

### 🔢 Port Numbers

* Port numbers identify specific services
* Range: **0 – 65535**

#### Common Ports:

* **80** → HTTP
* **443** → HTTPS
* **21** → FTP
* **22** → SSH
* **25** → SMTP
* **53** → DNS
* **110** → POP3
* **143** → IMAP

---

## 🔄 How It Works Together

1. A device is identified using an **IP address**
2. A **protocol** defines communication rules
3. A **port number** identifies the service
4. Logical ports handle the communication internally

## 💡 Key Takeaways

* Logical ports are virtual, not hardware
* Protocols define how systems communicate
* Port numbers identify specific services
* Many protocols are used in cybersecurity and networking
